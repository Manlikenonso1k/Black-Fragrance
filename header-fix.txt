### **The Global Layout Refactor Prompt**

> **"Perform a bulk refactor on all `.blade.php` files within `resources/views` (excluding the `layouts` directory). For every file found:**
>
> 1.  **Remove Boilerplate:** Delete all instances of `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`, `<header>`, and `<footer>` tags, including their closing tags and any content inside the original header/footer.
> 2.  **Extend Layout:** Add `@extends('layouts.apps')` to the very top of the file.
> 3.  **Define Content:** Wrap the remaining unique page content within a `@section('content')` and `@endsection` block.
> 4.  **Handle Metadata:** If a file contains a unique `<title>` tag, convert it to `@section('title', 'Title Text')`. 
> 5.  **Asset Management:** Move any page-specific `<style>` or `<script>` tags into `@push('styles')` or `@push('scripts')` blocks respectively.
>
> **Verify that no duplicate header/footer tags remain in the final files."**

---

### **Why this solves the "Old Header" issue**
* **Explicit Deletion:** It specifically instructs the AI to find and **delete** the old tags rather than just pasting the new layout code around them.
* **Exclusion Logic:** It tells the AI to ignore the `layouts` folder so it doesn't accidentally try to make `apps.blade.php` extend itself, which would cause an infinite loop.
* **Asset Preservation:** By using `@push`, it ensures that if you have a specific page (like a gallery or a contact form) that needs a special script, it won't break when the header is removed.
